üéØ Introduction to Engineering Optimization

What is Optimization?

Optimization is the scientific process of finding the best possible solution to a problem given a set of constraints. In engineering, this means:

  • Minimizing costs, weight, or energy consumption
  • Maximizing efficiency, profit, or strength
  • Finding the optimal balance between conflicting objectives

Why Study Optimization?

  • üèóÔ∏è Civil Engineering: Design cost-effective, safe structures
  • ‚úàÔ∏è Aerospace: Minimize weight while maintaining strength
  • ‚öôÔ∏è Mechanical: Maximize efficiency of machines
  • üîå Electrical: Optimize power distribution networks
  • üíº Operations: Allocate resources efficiently
üìå Key Principle: The maximum of a function f(X) occurs at the same location as the minimum of -f(X). This equivalence allows us to use minimization algorithms for all optimization problems.

üéõÔ∏è The Design Vector

The design vector is the mathematical representation of all the choices an engineer can make. It is the "DNA" of the design.

Design Variables vs. Preassigned Parameters

Design Variables (x‚ÇÅ, x‚ÇÇ, ...): Quantities the engineer can change to optimize performance. These go INTO the design vector.

Preassigned Parameters: Fixed quantities like material properties, safety factors, or prescribed dimensions. These stay OUTSIDE the design vector.

The Design Space

When we have n design variables, we create an n-dimensional Cartesian coordinate system called the Design Space. Each point in this space represents one complete design option:

  • 1 variable ‚Üí 1D Design Space (a line)
  • 2 variables ‚Üí 2D Design Space (a plane)
  • 3 variables ‚Üí 3D Design Space (a volume)
  • n variables ‚Üí n-D Design Space (a hyperspace)

üîí Design Constraints

Constraints are the limitations that define which designs are acceptable. They transform an unconstrained problem (where optimal solution is often trivial: zero size, minimum/maximum everything) into a realistic engineering problem.

Two Types of Constraints

1. Behavior (Functional) Constraints

These limit the performance or response of the system under load or operation.

Examples: - Stress induced ‚â§ Material yield strength - Deflection ‚â§ Maximum allowable deflection - Temperature ‚â§ Material melting point - Natural frequency > Operating frequency (avoid resonance)

2. Geometric (Side) Constraints

These limit the variables themselves, based on manufacturing, availability, or space limitations.

Examples: - 10 mm ‚â§ Shaft diameter ‚â§ 50 mm (available chuck sizes) - Fuel tank length ‚â§ Fuselage length (space envelope) - Number of teeth must be an integer - All dimensions must be positive

Mathematical Formulation

Inequality g‚±º(X) ‚â§ 0 Most common; defines a feasible region (one side of a boundary)
Equality h‚±º(X) = 0 Design must satisfy exact relationships (e.g., force balance)

The Feasible Region

The set of all points where all constraints are satisfied is called the Feasible Region. Visual understanding is critical:

  • Feasible Point: g(X) ‚â§ 0 (inside or on the boundary)
  • Infeasible Point: g(X) > 0 (outside the boundary, violates constraint)

Active vs. Inactive Constraints

Active Constraint: g(X*) = 0 (the design point sits exactly on the limit)

Inactive Constraint: g(X*) < 0 (the design point is safely inside the limit)

At the optimal solution, we focus on the active constraints‚Äîthey are the "bottlenecks" limiting further improvement.

üìà Single-Variable Optimization

Given a continuous function f(x), find the value x* that minimizes (or maximizes) f.

Analytical Methods (Calculus-Based)

Necessary Condition (Finding Stationary Points)

At a minimum or maximum, the derivative must be zero: f'(x*) = 0 Points where f'(x) = 0 are called STATIONARY POINTS (could be minimum, maximum, or inflection point)

Sufficient Condition (Determining the Type)

If f'(x*) = 0, then: - If f''(x*) > 0 ‚Üí x* is a LOCAL MINIMUM (valley) - If f''(x*) < 0 ‚Üí x* is a LOCAL MAXIMUM (peak) - If f''(x*) = 0 ‚Üí Test is inconclusive (check higher derivatives)

Numerical Methods (When Calculus Fails)

Elimination Methods

These methods narrow down the interval containing the minimum by eliminating regions where the minimum cannot be.

Method Concept Convergence Rate
Golden Section Divide interval by golden ratio (0.618), reuse one point each iteration Linear (most efficient elimination method)
Fibonacci Use Fibonacci sequence to divide interval, optimal if iteration count is fixed Linear
Dichotomous Place two points near center, eliminate half each iteration Linear
Exhaustive Evaluate at n equally-spaced points, pick the best Slow - only use for small problems

Interpolation Methods

These use the VALUES of f(x) at multiple points to fit a polynomial and estimate the minimum of that polynomial.

  • Quadratic Interpolation: Fit a parabola, find its minimum, use that as next test point
  • Cubic Interpolation: Fit a cubic, faster convergence if derivatives available

Gradient-Based Methods

When the derivative f'(x) can be calculated, these methods are much faster.

Newton's Method: x_{k+1} = x_k - f'(x_k) / f''(x_k) Secant Method (when f'' is hard to compute): Approximate f''(x) using finite differences of f' x_{k+1} = x_k - f'(x_k) ¬∑ (x_k - x_{k-1}) / (f'(x_k) - f'(x_{k-1}))

Convergence Comparison

Method Convergence Type Requirements When to Use
Elimination Linear (slow but steady) Just function values Non-smooth functions, black-box
Interpolation Super-linear (faster) Function values Smooth functions, robustness matters
Newton Quadratic (very fast) f'(x) and f''(x) Good starting point available
Practical Approach: In real software, these methods are often combined. Start with Golden Section to bracket the minimum reliably, then switch to Newton's method for final polishing.

‚úÖ Necessary and Sufficient Conditions for Optimality

These conditions form the mathematical foundation for identifying and verifying optimization solutions.

Understanding the Concepts

Necessary Condition: A condition that MUST be satisfied IF a solution is optimal. It's a screening test. Passing doesn't guarantee optimality, but failing disqualifies the point.

Sufficient Condition: A condition that GUARANTEES optimality IF it's satisfied. It's a confirmation test.

üî¢ Multivariable Optimization Without Constraints

When design variables are free to move in any direction (no physical limits), we have the cleanest optimization problem.

The Problem

Find X* = {x‚ÇÅ*, x‚ÇÇ*, ..., x‚Çô*}·µÄ which minimizes f(X) (No constraints, no limits)

Solution Process

  1. Necessary Condition: Set ‚àáf(X*) = 0 and solve for X*
  2. Sufficient Condition: Check the Hessian matrix H at X* to confirm it's a minimum

The Role of the Hessian

The Hessian matrix tells us the local curvature of the function:

  • Positive Definite (all eigenvalues > 0): Bowl-shaped ‚Üí Local minimum ‚úì
  • Negative Definite (all eigenvalues < 0): Dome-shaped ‚Üí Local maximum ‚úó (for minimization)
  • Indefinite (mixed eigenvalues): Saddle-shaped ‚Üí Saddle point ‚úó (not an extremum)

Unconstrained vs. Constrained

The optimal solution in unconstrained problems often lies in the interior of the search domain (far from boundaries), because there are no boundaries! In constrained problems, the optimum usually lies ON a constraint boundary. This fundamental difference requires different algorithmic approaches.

Practical Algorithms for Multivariable Unconstrained

Gradient Descent (Steepest Descent)

Algorithm: 1. Start with initial guess X‚ÇÄ 2. Calculate gradient ‚àáf(X_k) 3. Move in opposite direction (downhill): X_{k+1} = X_k - Œ±¬∑‚àáf(X_k) 4. Choose step size Œ± carefully (too large ‚Üí overshoot, too small ‚Üí slow) 5. Repeat until convergence (‚àáf ‚âà 0) Advantage: Simple, requires only first derivatives Disadvantage: Can be slow (zig-zagging near optimum)

Newton's Method (for multivariable)

Algorithm: X_{k+1} = X_k - H‚Åª¬π¬∑‚àáf(X_k) where H is the Hessian matrix and H‚Åª¬π is its inverse Advantage: Quadratic convergence (very fast) Disadvantage: Requires computing and inverting Hessian (expensive for large problems)

‚öñÔ∏è Multivariable Optimization with Equality Constraints

When design variables are related by exact equations (like conservation laws or geometric relationships).

The Problem

Minimize: f(X) Subject to: h‚±º(X) = 0, j = 1, 2, ..., m (m equality constraints, usually m < n)

Key Insight: Degrees of Freedom

  • n variables total
  • m constraints (each removes one degree of freedom)
  • Results in (n - m) independent variables

If n = m, the problem is fully determined (likely no optimization to do). If n < m, the problem is over-constrained (no solution).

Three Solution Methods

Method 1: Direct Substitution

Idea: Solve constraints for m variables, substitute into objective to eliminate constraints

Problem: Minimize f(x‚ÇÅ, x‚ÇÇ) subject to x‚ÇÅ¬≤ + x‚ÇÇ¬≤ = 1 Solution: Substitute x‚ÇÇ = ‚àö(1 - x‚ÇÅ¬≤) into f Result: Minimize f(x‚ÇÅ, ‚àö(1 - x‚ÇÅ¬≤)) with respect to x‚ÇÅ only (Now it's unconstrained in one variable!)

üîó Method of Lagrange Multipliers

The most powerful and elegant method for handling equality constraints in optimization.

Core Idea

Instead of enforcing constraints directly, we "absorb" them into the objective function using penalty multipliers. At optimality, the gradient of the (unconstrained) modified function is zero.

The Lagrange Function

L(X, Œª) = f(X) + Œ£ Œª‚±º¬∑h‚±º(X) where: - f(X) is the original objective - h‚±º(X) are the constraints - Œª‚±º are Lagrange multipliers (one per constraint) The multipliers are "prices" - how much we're willing to "pay" to violate each constraint.

At the optimum, the gradient of L with respect to ALL variables must be zero:

‚àÇL/‚àÇx·µ¢ = ‚àÇf/‚àÇx·µ¢ + Œ£ Œª‚±º¬∑(‚àÇh‚±º/‚àÇx·µ¢) = 0, i = 1, ..., n ‚àÇL/‚àÇŒª‚±º = h‚±º(X) = 0, j = 1, ..., m This gives (n + m) equations for (n + m) unknowns

At optimality: -‚àáf = Œ£ Œª‚±º¬∑‚àáh‚±º This means the negative gradient of the objective (direction of steepest descent) is a linear combination of the constraint gradients (normals to the constraint surfaces). Physically: The objective function contour is TANGENT to the constraint surface at the optimum.

The Lagrange multiplier Œª‚±º* represents:

Œª‚±º* = ‚àÇf*/‚àÇb_j

The rate of change of the optimal objective value f* with respect to the constraint right-hand side b‚±º. Example: If a constraint is "Available Material = 100 kg" and Œª* = 5, then relaxing it to 101 kg would improve the objective by approximately 5 units.

Solving the System

  1. Construct L = f(X) + Œ£ Œª‚±º¬∑h‚±º(X)
  2. Take partial derivatives: ‚àÇL/‚àÇx·µ¢ and ‚àÇL/‚àÇŒª‚±º
  3. Set all derivatives to zero
  4. Solve the resulting (n + m) nonlinear equations for X* and Œª*
  5. Check sufficiency using the bordered Hessian (advanced)

Example: Minimize f(x‚ÇÅ, x‚ÇÇ) = x‚ÇÅ¬≤ + x‚ÇÇ¬≤ subject to x‚ÇÅ + x‚ÇÇ = 1

Step 1: Construct Lagrangian

h(x‚ÇÅ, x‚ÇÇ) = x‚ÇÅ + x‚ÇÇ - 1 = 0 L(x‚ÇÅ, x‚ÇÇ, Œª) = x‚ÇÅ¬≤ + x‚ÇÇ¬≤ + Œª(x‚ÇÅ + x‚ÇÇ - 1)

‚àÇL/‚àÇx‚ÇÅ = 2x‚ÇÅ + Œª = 0 ‚àÇL/‚àÇx‚ÇÇ = 2x‚ÇÇ + Œª = 0 ‚àÇL/‚àÇŒª = x‚ÇÅ + x‚ÇÇ - 1 = 0 Three equations, three unknowns

Key Theorems

Theorem 1 (Necessary Conditions): If X* is a local minimum, it must satisfy the Lagrange conditions. Theorem 2 (Sufficient for Convex Problems): If f and h are convex (or linear), the Lagrange conditions are sufficient for a global minimum.

Geometric Interpretation

The Lagrange multiplier method has a clear geometric meaning:

üìê Lagrange Geometry: At the constrained optimum, the objective function gradient ‚àáf is parallel to the constraint gradient ‚àáh. This means the level curve of f is tangent to the constraint curve‚Äîthey have the same normal direction. The multiplier Œª represents the rate of change of the optimal objective value with respect to constraint relaxation (shadow price). If Œª = -1, relaxing the constraint by 1 unit improves the objective by 1 unit.

‚õî Multivariable Optimization with Inequality Constraints

The most realistic engineering problems have BOTH equality and inequality constraints. We focus on inequalities here.

The Problem

Minimize: f(X) Subject to: g‚±º(X) ‚â§ 0, j = 1, 2, ..., m (These are the REALISTIC constraints - not exact requirements, but limits)

Key Concepts

Active vs. Inactive Constraints

At the optimum X*:

  • Active: g‚±º(X*) = 0 (exactly on the limit)
  • Inactive: g‚±º(X*) < 0 (safely inside the limit)

For inactive constraints, we can ignore them locally (they don't affect the immediate search direction). For active constraints, they are like equality constraints - they control movement.

Complementary Slackness

Either a constraint is active OR its multiplier is zero (but not both in certain cases):

Œª‚±º ¬∑ g‚±º(X) = 0 for all j Interpretation: - If Œª‚±º > 0, then g‚±º(X) = 0 (constraint is active) - If g‚±º(X) < 0, then Œª‚±º = 0 (constraint is inactive)

The Slack Variable Transformation

To convert inequalities to equalities (enabling Lagrange multipliers), we introduce slack variables:

Original: g‚±º(X) ‚â§ 0 Modified: g‚±º(X) + y‚±º¬≤ = 0 where y‚±º¬≤ represents "slack" (must be ‚â• 0, so it's squared) If y‚±º = 0, then g‚±º(X) = 0 (constraint is active) If y‚±º ‚â† 0, then g‚±º(X) < 0 (constraint is inactive)

Geometric Intuition

Visualizing how constraints affect optimization:

üó∫Ô∏è Feasible Region Geometry: Inequality constraints define a feasible region (shaded area). Active constraints (g = 0) form the boundary; inactive constraints (g < 0) are satisfied with slack. The optimal solution usually sits where multiple constraints meet (are active simultaneously). Active constraints are the bottlenecks‚Äîthey limit further improvement. Inactive constraints don't affect the optimal value.
Think of the inequality constraints as defining a "feasible region" (like a city) bounded by constraint surfaces (like city walls). Your objective function creates a landscape (like a terrain). You want to reach the lowest point in the city. If the lowest terrain point is OUTSIDE the city walls, the optimum is where an objective contour is TANGENT to a city wall (at least one active constraint). If the lowest terrain point is INSIDE the city, that's where you want to go (no constraints matter).

üéØ The Kuhn-Tucker (K-T) Conditions

The foundation of modern nonlinear programming. These conditions unify optimization with inequality constraints.

Historical Context

Developed by Kuhn and Tucker (1951) as a generalization of Lagrange multipliers to handle inequalities. Essential for both theoretical analysis and practical algorithms.

The Four K-T Conditions

For the problem: Minimize f(X) subject to g‚±º(X) ‚â§ 0 At a point X* to be optimal, it must satisfy: 1. STATIONARITY (Gradient Balance): ‚àáf(X*) + Œ£ Œª‚±º‚àág‚±º(X*) = 0 2. PRIMAL FEASIBILITY (Constraints Satisfied): g‚±º(X*) ‚â§ 0 for all j 3. COMPLEMENTARY SLACKNESS (The Switch): Œª‚±º ¬∑ g‚±º(X*) = 0 for all j 4. DUAL FEASIBILITY (Multiplier Signs): Œª‚±º ‚â• 0 for all j (for minimization with ‚â§ constraints)

Interpreting Each Condition

1. Stationarity

The negative gradient of the objective must be a weighted sum of constraint gradients:

-‚àáf = Œ£ Œª‚±º‚àág‚±º If no constraints are active (all Œª = 0), this reduces to ‚àáf = 0 (standard unconstrained optimality). If constraints are active, -‚àáf points "into" the feasible region (weighted by constraint importance Œª‚±º).

2. Primal Feasibility

Simple: The solution must obey all constraints. No point in finding an "optimal" infeasible design.

3. Complementary Slackness

The "switching" condition that ties multipliers to active constraints:

Think of a spring with a natural length. Either: (a) The spring is taut (Œª > 0) and stretched exactly to length 0 (g = 0), OR (b) The spring is slack (g < 0), so there's no tension (Œª = 0) Both being nonzero simultaneously would mean a slack spring is exerting tension - impossible!

4. Dual Feasibility

Multipliers must be non-negative for ‚â§ constraints in a minimization problem:

The "force" from constraints must push INTO the feasible region (the direction of -g). Negative Œª would mean constraints are pulling the solution OUT of the feasible region - violating optimality.

Step 1: Form the Lagrangian: L = f(X) + Œ£ Œª‚±ºg‚±º(X)

Step 2: Write K-T equations (stationarity + complementarity)

Step 3: Since you don't know which constraints are active, try all 2^m cases:

  • Case 2: g‚ÇÅ active (Œª‚ÇÅ > 0, g‚ÇÅ = 0), others inactive (Œª‚ÇÇ = Œª‚ÇÉ = ... = 0)
  • ... and so on

Constraint Qualification

K-T conditions are necessary ONLY if Constraint Qualification holds:

  • The gradients of active constraints must be linearly independent
  • OR Slater's condition: There exists a point strictly inside all inequality constraints

When K-T is Sufficient (Guaranteed Optimality)

For CONVEX PROGRAMMING problems: - Objective f is convex - All constraints g‚±º are convex - Equality constraints (if any) are linear The K-T conditions are NECESSARY AND SUFFICIENT for a GLOBAL minimum!

Applications in Engineering

  • Circuit Design: Minimize power subject to timing constraints
  • Portfolio Optimization: Maximize return subject to risk constraints
  • Process Control: Optimize production subject to safety/environmental constraints
